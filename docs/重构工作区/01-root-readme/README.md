# 任务 01：重写根 README

## 目标

让第一次打开仓库的开发者在三分钟内理解项目定位，在十分钟内找到可运行示例。README 是产品入口，不是架构论文或完整 API 手册。

## 修改范围

- 只负责根目录 `README.md`。
- 文档导航链接可以指向其他任务计划创建的目标路径；不要代写其他文件。

## README 必须回答

1. 这是什么：面向 pi 的单 Agent 串行、可循环图编排 SDK。
2. 适合什么：多阶段生成/检查/修改、工具受控执行、代码与 Agent 混合流程。
3. 不是什么：不是并行 Agent 框架，不提供会话持久恢复，不承诺生产稳定。
4. 如何安装和创建 extension。
5. 如何定义一张最小图并运行。
6. 下一步去哪里阅读。

## 必须纠正的旧内容

- 删除“所有扩展点都是纯函数”。
- 不再把 call 边界描述成唯一子图隔离方式。
- 不把兼容 frame 字段写成固定要求。
- 更新项目结构，或者彻底删除容易漂移的完整文件树。
- 将版本定位写为首个测试版本/alpha，不宣称生产稳定。
- 不把 demo extension 安装方式放在 library 使用方式之前。

## 推荐结构

```text
标题 + 一句话定位
状态徽标/alpha 提示
为什么使用
核心能力（最多 7 条）
安装
最小示例
子图边界简表
当前限制
文档导航
License
```

## 示例约束

- 示例只使用一个 code 节点、一个 END 边和一次注册。
- frame 示例使用业务字段，如 `{ outcome: completion.result }`。
- 不在第一个示例中引入 Mechanism、skill、renderer 或 delegate。
- 所有导入必须来自当前 `src/index.ts` 的真实导出。

## 验收

- README 控制在约 250–400 行以内，优先更短。
- 从零阅读不需要理解 scope、visit、projection、compaction。
- 复制最小示例后能够通过 typecheck。
- 当前支持和不支持的能力都有明确列表。
