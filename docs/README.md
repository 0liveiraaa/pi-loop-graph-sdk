## docs使用说明

  为了便于文档管理,我们将一个idea的生命周期分为,设想->待实现->已实现/废弃
  因而docs文件夹下建立了**设计**,**计划**,**形态**,**归档**四个部分

1. 设计(design):其中设计部分主要关注当前正在思考idea的初步规划
2. 计划(plan):关注如何实现idea,是agent动手的基本蓝图和重要参考
3. 形态(待定):是当前实现结果,有详细的实现说明报告,以帮助agent快速理解当前项目情况,因而需要及时更新
4. 归档(archived),当以上三个文件夹内文件**过时**时,进入此文件夹,并以20xx/xx/xx_name_design/plan/.../的格式命名,并在文件夹中用readme开头写明归档原因

#### 归档条件:

- 设计:被决定采用或废弃,若被废弃需要在文件开头写明废弃原因
- 计划:被实施完成或终止,若被终止需要在文件开头写明终止原因
- 形态:被迭代,需要说明被迭代原因
